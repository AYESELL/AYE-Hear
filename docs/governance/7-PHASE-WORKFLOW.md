# 7-Phase Workflow

AYE Hear uses the shared AYE delivery discipline with an explicit architecture gate before implementation.

1. PREP
   Load the task, clarify acceptance criteria and confirm repository scope.

2. CONTEXT AND DESIGN
   Read ADRs, review product foundation, align on risks and obtain architect approval.

3. IMPLEMENT
   Build the minimal correct solution inside approved boundaries.

4. TEST
   Add unit and integration tests, target at least 75 percent coverage.

5. VALIDATE
   Check privacy, offline-first behavior, performance and speaker-confidence handling.

6. REVIEW
   Run code review and verify quality gates.

7. RELEASE READY
   Prepare installer, release notes and deployment readiness.

8. COMPLETE
   Update documentation, ADRs and task implementation notes before closing the task.
